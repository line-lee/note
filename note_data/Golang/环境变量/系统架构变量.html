🏗️ 系统架构与交叉编译变量

💡 核心概念：Go 支持跨平台编译，通过设置这些变量可以生成不同操作系统和架构的可执行文件。

GOOS - 目标操作系统

export GOOS=linux     # Linux
export GOOS=windows   # Windows
export GOOS=darwin    # macOS
export GOOS=freebsd   # FreeBSD
export GOOS=android   # Android
值说明
linuxLinux 系统
windowsWindows 系统
darwinmacOS / iOS
freebsdFreeBSD
netbsdNetBSD
openbsdOpenBSD
plan9Plan 9
solarisSolaris
androidAndroid
iosiOS
jsWebAssembly

GOARCH - 目标架构

export GOARCH=amd64    # 64位 x86
export GOARCH=386      # 32位 x86
export GOARCH=arm64    # 64位 ARM
export GOARCH=arm      # 32位 ARM
值说明
amd6464位 x86 架构（最常见）
38632位 x86 架构
arm6464位 ARM 架构（树莓派、M1/M2/M3）
arm32位 ARM 架构
ppc6464位 PowerPC
s390xIBM S/390x
wasmWebAssembly
riscv6464位 RISC-V

常见组合示例

# Linux 64位
GOOS=linux GOARCH=amd64 go build

# Windows 64位
GOOS=windows GOARCH=amd64 go build

# macOS (Intel)
GOOS=darwin GOARCH=amd64 go build

# macOS (Apple Silicon)
GOOS=darwin GOARCH=arm64 go build

# 树莓派 (ARM 32位)
GOOS=linux GOARCH=arm GOARM=7 go build

# WebAssembly
GOOS=js GOARCH=wasm go build -o main.wasm

GOAMD64 - AMD64 微架构级别

控制 AMD64 架构的代码生成优化级别（Go 1.17+）。

export GOAMD64=v1   # 兼容所有 x86-64 处理器
export GOAMD64=v2   # 需要 SSE4.2、POPCNT 等
export GOAMD64=v3   # 需要 AVX、AVX2、BMI1/BMI2 等
export GOAMD64=v4   # 需要 AVX512F、AVX512BW 等
🔍 查看支持的组合：运行 go tool dist list 查看所有支持的操作系统和架构组合。

交叉编译完整示例

#!/bin/bash
# 为多个平台编译

platforms=(
    "linux/amd64"
    "linux/arm64"
    "windows/amd64"
    "darwin/amd64"
    "darwin/arm64"
)

for platform in "${platforms[@]}"; do
    IFS='/' read -r GOOS GOARCH <<< "$platform"
    output="myapp-${GOOS}-${GOARCH}"
    if [ $GOOS = "windows" ]; then
        output+='.exe'
    fi

    echo "Building for $GOOS/$GOARCH..."
    GOOS=$GOOS GOARCH=$GOARCH go build -o "$output" .
done
