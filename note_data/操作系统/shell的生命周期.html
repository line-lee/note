🐚 Shell 生命周期核心概念 fork · execve · 信号
🔄 命令执行模式：串行 vs 并行 父进程 PID 存活保证
📌 串行执行 (; 或 && 或 ||)
Shell (PID=1000)
父进程持续存活
→
fork()
创建子进程
→
execve()
替换为命令
→
wait()
等待子进程退出
💡 父进程存活保证：Shell 通过 wait() 阻塞等待子进程结束，父进程 PID 始终保持不变。
⚡ 并行执行 (& 后台运行)
Shell (PID=1000)
父进程持续存活
→
fork()
创建子进程
→
execve()
命令后台运行
Shell (PID=1000)
不 wait()，立即返回
→
子进程独立运行
父进程不阻塞
💡 父进程存活保证：Shell 调用 fork() 后不调用 wait()，父进程 PID 始终保持不变，子进程退出时通过 SIGCHLD 信号异步通知。
🌲 进程树结构演示 PID 关系图
├─ init (PID=1) ─── 系统根进程
│ └─ bash/zsh (PID=1000) ─── Shell 父进程 (始终存活)
│ ├─ ├─ ls (PID=1001) ─── 串行执行，wait() 等待退出
│ ├─ ├─ grep (PID=1002) ─── 串行执行
│ ├─ ├─ sleep 100 & (PID=1003) ─── 后台并行，父进程不等待
│ └─ └─ python server.py & (PID=1004) ─── 后台并行
