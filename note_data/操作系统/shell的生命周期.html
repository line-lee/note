🐚 Shell 生命周期核心概念 fork · execve · 信号
🔄 命令执行模式：串行 vs 并行 父进程 PID 存活保证
📌 串行执行 (; 或 && 或 ||)
Shell (PID=1000)
父进程持续存活
→
fork()
创建子进程 PID=1001
→
execve()
替换为命令1
→
wait()
Shell 等待子进程退出
Shell (PID=1000)
继续执行
→
fork() → execve()
执行命令2
→
wait()
再次等待
💡 父进程存活保证：Shell 通过 wait() 系统调用阻塞等待子进程结束，父进程 PID 始终保持不变 (1000)，子进程结束后由 init (PID=1) 接管僵尸进程回收。
⚡ 并行执行 (& 后台运行)
Shell (PID=1000)
父进程持续存活
→
fork() #1
创建子进程A (PID=1001)
→
execve()
命令1 后台运行
→
Shell (PID=1000)
不 wait()，立即返回
→
fork() #2
创建子进程B (PID=1002)
→
execve()
命令2 后台运行
Shell (PID=1000)
继续接受用户输入
⏰
子进程A/B 独立运行
父进程不阻塞
💡 父进程存活保证：Shell 调用 fork() 后不调用 wait()，子进程成为孤儿进程会被 init (PID=1) 收养，但父进程 Shell (PID=1000) 始终保持存活，继续处理后续命令。子进程退出时通过 SIGCHLD 信号异步通知父进程。
🌲 进程树结构演示 PID 关系图
├─ init (PID=1) ─── 系统根进程
│ └─ bash/zsh (PID=1000) ─── Shell 父进程 (始终存活)
│ ├─ ├─ ls (PID=1001) ─── 串行执行，wait() 等待退出
│ │ └─ [已退出，僵尸状态由父进程回收]
│ ├─ ├─ grep (PID=1002) ─── 串行执行
│ │ └─ [已退出]
│ ├─ ├─ sleep 100 & (PID=1003) ─── 后台并行，父进程不等待
│ │ └─ [独立运行，父进程继续]
│ └─ └─ python server.py & (PID=1004) ─── 后台并行
│ └─ [独立运行]
✨ 关键：无论串行还是并行，Shell 父进程 (PID=1000) 始终保持存活，直到用户 exit 或收到终止信号。
📡 信号处理机制 SIGCHLD · SIGINT · SIGTERM
🔧 子进程退出时的父进程存活保证：
1️⃣ 子进程终止时，内核发送 SIGCHLD 信号给父进程。
2️⃣ Shell 的信号处理函数调用 waitpid() 回收子进程资源，避免僵尸进程。
3️⃣ 父进程 Shell 本身不会因为子进程退出而退出，除非显式收到 SIGINT (Ctrl+C) 或 SIGTERM。
4️⃣ 若父进程在子进程退出前死亡，子进程被 init (PID=1) 收养，继续运行。
